# Plan: RR-15

## Work Item
**Title:** Connect to Microsoft Foundry agent

## Overview
RR-15 connects the Express server (scaffolded in RR-13) to a published Microsoft Foundry agent. Authentication uses `DefaultAzureCredential` from `@azure/identity` with the `https://ai.azure.com/.default` scope — the token is passed as a bearer header in a direct `fetch` call to `FOUNDRY_PROJECT_ENDPOINT`. The `POST /api/chat` endpoint accepts a message, calls the Foundry Responses API, and returns the agent's reply along with any citations. Frontend wiring follows in a subsequent branch.

## Prerequisites
- The Foundry agent must be **published** in the Foundry portal before the Express server can call it. Unpublished agents are not reachable via the SDK.
- For local development, authentication is handled via `az login` — no static API key is required.
